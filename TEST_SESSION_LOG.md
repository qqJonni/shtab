## Список найденных багов и внесённых правок

### Правка #1 — Список пользователей не отображается
**Проблема:** При нажатии «Пользователи» под admin — страница не открывалась (вела на заглушку дашборда).
**Решение:** Создан `routes/admin.py` с `/admin/users` — список пользователей с подтверждением, блокировкой, сменой роли. Навигация обновлена.

### Правка #2 — Удаление пользователя
**Запрос:** Администратор должен иметь возможность удалить пользователя.
**Решение:** Добавлен роут `POST /admin/users/<id>/delete`. Первоначально падал с FK constraint — переделан на двухшаговое удаление (страница связей → подтверждение → откреплние всех связей).

### Правка #3 — Удаление организации
**Запрос:** Аналогично пользователям — удаление организации с показом связей.
**Решение:** Роут `/organizations/<id>/delete` с двухшаговым подтверждением. При удалении: пользователи открепляются, этапы переходят в «поиск подрядчика».

### Правка #4 — Защита admin от блокировки
**Запрос:** Ни один пользователь не может заблокировать администратора.
**Решение:** Проверка роли в `user_block`, кнопка блокировки скрыта для admin-пользователей.

### Правка #5 — Создание организации
**Запрос:** Администратор/руководитель могут создать организацию с реквизитами и привязать подрядчиков.
**Решение:** Роут `/organizations/add` с полной формой (название, тип, адрес, телефон, email, ИНН/КПП/ОГРН/ОКПО, представитель). Чекбоксы для привязки подрядчиков. Поле email добавлено через миграцию.

### Правка #6 — Удаление организации для руководителя
**Запрос:** Руководитель тоже должен удалять организации.
**Решение:** `@role_required('admin', 'manager')` вместо только `admin`.

### Правка #7 — ПТО загружает документы этапа
**Запрос:** Инженер ПТО должен иметь право подгружать документы к этапу.
**Решение:** Добавлен `pto` в `_can_upload_doc()`.

### Правка #8 — Убрано поле «Расценка» из формы подэтапа
**Запрос:** При создании подэтапа не нужно поле «Расценка».
**Решение:** Поле убрано из `substage_form.html`, колонки перестроены.

### Правка #9 — Убраны столбцы «Расценка», «Сумма» и «Итого» из таблицы подэтапов
**Запрос:** В окне этапов убрать финансовую информацию.
**Решение:** Из `stage_detail.html` убраны столбцы «Расценка», «Сумма» и строка «Итого по этапу».

### Правка #10 — Убраны «Расценка» и «Сумма» с карточки подэтапа
**Запрос:** У подрядчика в окне подэтапа убрать столбцы расценки и суммы.
**Решение:** Из `substage_detail.html` убраны блоки «Расценка» и «Сумма».

### Правка #11 — Ссылка-назад для подрядчика
**Баг:** На странице этапа ссылка вверху вела на объект → 403 для подрядчика.
**Решение:** Для contractor ссылка ведёт на «Мои этапы» вместо страницы объекта.

### Правка #12 — Тип документа «Ценовой документ»
**Запрос:** Добавить пункт «Ценовой документ» в загружаемые документы этапа.
**Решение:** Добавлен `price_doc` в `DOC_TYPE_LABELS`, обновлён CHECK constraint через миграцию.

### Правка #13 — ПТО и подрядчик удаляют любые документы этапа
**Запрос:** Оба могут удалять прикреплённые документы вне зависимости от загрузившего.
**Решение:** Обновлена логика `stage_doc_delete` — проверка по роли, не по `uploaded_by`.

### Правка #14 — Переименование «Состав работ»

### Правка #15 — Индикатор просрочки подэтапов

**Решение:** Красный бейдж «Просрочен» + дата красным в таблице и на карточке подэтапа.
